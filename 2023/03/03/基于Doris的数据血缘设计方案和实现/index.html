基于Doris的数据血缘设计方案和实现

数据血缘简介：

在数据仓库的建设中，必然会有各个表间的纷繁交错的表数据依赖关系，所以理清数据的流向关系就显得尤为重要。而数据血缘就是为了完成这项任务，是在数据仓库的基础上，理解数据从哪里来到哪里去，对数仓的治理提供依据，譬如影响性分析，跨层引用率，引用热度，调度依赖等等。

数据血缘实现方案：

收集Doris审计日志，修改doris源代码使得审计日志中的每段sql有分行符用于拆分

Doris Audit Plugin，开发Doris插件，用于收集Doris审计日志，做数据格式化，最后sink Kafka dmp_ audit_ log Topic。

Audit Log ETL Service微服务进行数据清洗，乱序数据排序，产生血缘的sql执行记录Sink Kafka dmp_lineage Topic(产生血缘的 SQL 执行记录)

SQL AST Parser：利用Druid解析AST语法树递归获得数据血缘关系，结果插入到Nebula图数据库
